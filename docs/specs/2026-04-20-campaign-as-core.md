# Kampagne als zentrales Element

**Status:** Draft
**Datum:** 2026-04-20

## Kernidee

Die Kampagne ist das führende, organisierende Element im CIN Radar. Alles passiert im Kontext einer Kampagne. Sie ist der Workspace, in dem der gesamte Innovation-Prozess abläuft.

## Flow

1. **Kampagne erstellen** — Thema/Fragestellung definieren, Zeitraum, Teilnehmer einladen
2. **Signale erfassen** — Teilnehmer und AI Scout sammeln Signale innerhalb der Kampagne
3. **Cluster bilden** — AI gruppiert Signale automatisch + User kann manuell clustern
4. **Cluster → Trends** — Review-Workflow: Cluster wird zum vorausgefüllten Trend-Steckbrief
5. **Visualisierung/Stream** — Live-Überblick was reinkommt, wie sich Cluster bilden, welche Trends entstehen

## Stream / Grafik

Die Kampagne zeigt einen visuellen Stream/Workflow:
- Eingehende Signale (links/oben)
- Wie sie sich zu Clustern verdichten (Mitte)
- Welche Trends daraus entstehen (rechts/unten)
- Quasi ein lebendiger Trichter/Flow der den Fortschritt zeigt

Dieser Stream könnte das mächtigste Feature der App sein — er macht den Innovations-Prozess sichtbar und greifbar.

## Was man in einer Kampagne machen kann

- Signale hinzufügen (manuell, URL, PDF, AI Scout)
- Cluster sehen, manuell erstellen, Signale zuordnen
- Cluster als Trend reviewen → vorausgefüllter Steckbrief
- Teilnehmer verwalten
- Stream/Visualisierung des gesamten Prozesses sehen
- Filterung nach Stage (Scout → Cluster → Validate → Rate)

## Offene Design-Fragen

- Ist ALLES kampagnen-gebunden, oder gibt es auch Signale/Trends ausserhalb von Kampagnen?
- Kann ein Signal/Trend in mehreren Kampagnen vorkommen?
- Stream-Visualisierung: Sankey-Diagramm? Timeline? Kanban-Flow? Was stellt man sich vor?
- Wie funktioniert die Teilnehmer-Einladung? (Link teilen, Email, Login?)
- Soll der Prozess-Fortschritt (Scout→Cluster→Validate→Rate) pro Kampagne getrackt werden?

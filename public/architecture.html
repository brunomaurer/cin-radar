CIN Radar · Systemarchitektur

Stand: 24.04.2026 · Prototyp · cin-radar.vercel.app

CIN Radar ist ein Trend- und Innovations-Radar-Tool, gebaut als Single-Page-Application mit Vite + React und deployed auf Vercel. AI-Features (Chat, Artefakt-Generierung, Similarity-Ranking, Text-to-Speech, Bild-Generierung) laufen über Vercel Serverless Functions gegen Anthropic Claude, OpenAI und Pollinations. Persistenz via Upstash Redis.

Wichtige Datenflüsse

MVP-Coach Chat

  1. Nutzer klickt in Initiative-Detail in den Chat-Input
  2. Frontend POST /api/chat mit messages[] + Brief-Kontext
  3. Function sendet an Anthropic Sonnet 4.6 mit System-Prompt
  4. Claude antwortet (~2-5s), Function returns JSON
  5. Frontend appended Assistant-Message, speichert Chat-History via PUT /api/concepts?id=…

Audio Briefing (MP3)

  1. Nutzer wählt Trends im Explorer, klickt Aktion → "Audio Briefing"
  2. POST /api/voice-script → Claude Sonnet schreibt flüssigen Vorlesetext (~2200 Zeichen)
  3. POST /api/voice-tts mit Script → OpenAI TTS tts-1 liefert MP3-Binärstream
  4. Frontend konvertiert Blob → Data-URL, öffnet neues Fenster mit Player + Download-Button

Management Summary (PDF)

  1. Nutzer wählt Trends, klickt Aktion → "Management Summary als PDF"
  2. POST /api/summary → Claude Sonnet generiert strukturiertes JSON (Executive / Insights / Recommendations / Opportunities / Risks)
  3. Frontend rendert HTML-Report mit CSS für Print
  4. window.print() → Browser-Dialog "Als PDF speichern"

Trend-Relations (Analytics)

  1. Nutzer öffnet /analytics/relations
  2. Für jeden Trend parallel POST /api/relations mit Kandidatenliste
  3. Server prüft Redis-Cache (Signatur aus Kandidaten-IDs). Miss → Claude Haiku, Hit → sofort zurück (TTL 24h)
  4. Frontend dedupliziert Kanten, rendert Force-Graph / Matrix / Chord via SVG

Trend erfassen

  1. Nutzer klickt "Neuer Trend" im Header
  2. NewTrendDialog: Titel, Dimension, Horizont, Stage, Rating-Slider, Tags, Summary
  3. POST /api/trends → Redis trend:<nanoid> + trends:index
  4. Navigate zu /trend/<id>, Explorer zeigt den neuen Trend

MVP-Artefakte generieren

  1. Nutzer füllt Brief aus, klickt "Artefakte generieren"
  2. Frontend ruft /api/generate sechs mal hintereinander — einer pro Artefakt-Typ (CLAUDE.md, PRD, Tech, Deck, Prompt, Email)
  3. Pro Call: Claude Sonnet 4.6, ~10-15s, max 2048 Tokens
  4. Frontend zeigt Progress, speichert nach jedem Artefakt via PUT /api/concepts?id=…

Deployment-Flow

Env-Variablen werden in Vercel → Settings → Environment Variables gepflegt. Änderungen an Env-Vars erfordern einen manuellen Redeploy.

Env-VariablePflicht für
ANTHROPIC_API_KEYAlle Claude-Aufrufe (Chat, Generate, Summary, Relations, etc.)
OPENAI_API_KEYAudio-Briefing MP3-Generierung
REDIS_URLPersistenz (automatisch gesetzt durch Upstash Marketplace Integration)

Sicherheits-Status (Prototyp)

Aktuell sind die API-Endpoints öffentlich — jeder mit der URL kann API-Credits verbrauchen. Für einen produktiven Betrieb wären folgende Schritte nötig:

  • offen Rate-Limiting (z.B. via Upstash Ratelimit SDK)
  • offen Authentifizierung (Password-Gate oder OAuth via Clerk/Supabase)
  • offen User-Scoped Daten statt shared Redis
  • da HTTPS (über Vercel default)
  • da API-Keys server-seitig (nie im Frontend)
